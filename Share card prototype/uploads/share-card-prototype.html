设计要点

画布：放弃 4:5，改用竖版 2:3（900×1350px），保留原高度 1350px，宽度与底部信息条对齐；设计稿以 500×750px 预览，导出时按 1.8× 渲染。

背景：完整版以海报放大、高斯模糊（40px）、压暗 45% 作为 full-bleed 氛围底色；海报跨域失败时使用深蓝渐变（#1a1a2e → #16213e → #0f3460）保持票根氛围。

五区布局（参考豆瓣）：① 左上海报；② 右上标题 + 评分/日期/类型/话数等基本信息；③ 中部简介；④ 底部二维码 + 扫码文案；⑤ 右下角品牌标识 + 条目 ID。

海报：圆角 24px，与卡片圆角统一，2:3 比例，宽度约 176px（设计稿），导出时约 317px；带柔和投影。

标题：中文名为主标题，允许两行；日文名为副标题，颜色较浅。

信息行：评分（珊瑚粉大号数字）+ 五星 + 人数；右侧日期/类型/话数使用等宽字体，中间用 1px 半透明线分隔。Staff 仅展示关键项，从 API 的 infobox 字段按平台取优先 key：动画取「导演 / 动画制作 / 原作」、漫画/书籍取「作者 / 出版社」、游戏取「开发 / 发行」等，最多 2 行，避免像浏览器预览卡片 .info.tip 那样把所有人员堆在一起。

简介：位于海报与 Staff 信息下方，3 行，左对齐，浅灰，溢出省略号，不再叠加渐变遮罩。

底部：浅灰信息条与卡片等宽（通过负 margin 满宽拼接），左侧二维码，右侧 bangumi.tv 品牌文字 + 条目 ID。

字体栈：Noto Sans JP / 思源黑体（Noto Sans SC / PingFang SC / Microsoft YaHei）负责标题与正文；JetBrains Mono / SF Mono 负责评分、日期、ID。
